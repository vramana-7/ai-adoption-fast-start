# Discovery-to-Program Mapping Rules

These rules convert discovery findings into specific 4-week plan adjustments.

## Rule Set

1. If confidence is low and usage is sporadic:
- Increase Week 1 guided reps.
- Keep Week 2 prompts tightly scaffolded.
- Add daily 10-minute practice prompts.

2. If governance risk is high:
- Add explicit do/don't boundaries in Week 1.
- Use only low-risk internal examples in Weeks 2-3.
- Require manager checkpoint before Week 4 workflow conversion.

3. If managers are not engaged:
- Add a weekly manager action checklist.
- Include quick review rituals (10 minutes per week).
- Keep success metrics visible and simple.

4. If team time is constrained:
- Reduce exercise volume and increase relevance.
- Focus on one pilot workflow only.
- Prioritize completion over breadth.

5. If resistance is based on quality concerns:
- Add output verification steps in every week.
- Track before/after quality measures.
- Include "human review required" guidance in exercises.
- Assign a metric owner for each weekly checkpoint.

6. If workflow fit is unclear:
- Run two mini pilots in Week 2.
- Select one winner for Weeks 3-4.
- Document why the selected workflow has highest adoption potential.

## Minimum Output Requirements

Every generated 4-week program must include:
- one clear business outcome
- one primary pilot workflow
- three confidence checkpoints
- one before/after workflow conversion
- two measurable success signals
- weekly checkpoints with metric, target, owner, and evidence artifact
- baseline and week-4 metrics with named owners and data sources
